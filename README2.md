## Project Architecture

The AI Search Assistant follows a simple architecture where the user interacts with a web-based chatbot interface built using Flask, HTML, and Tailwind CSS. When the user enters a question, the request is sent to the Flask backend application. The backend then uses LangChain to manage the workflow between internet search and the local AI model. First, the application performs a real-time internet search using DuckDuckGo Search to collect relevant information related to the user query. The search results, along with the original question, are then sent to the Ollama server, which runs the Phi3 local language model on the user's laptop. The AI model processes the combined information and generates an intelligent response. Finally, the generated response is returned to the Flask application and displayed back to the user in the chatbot interface. This architecture enables the application to provide real-time AI-powered responses while running completely on a local machine without relying on paid cloud AI APIs.
